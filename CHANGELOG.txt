## Changelog

### v3.2.0 Updates (March 2026)

**Shared User Token Recovery (Plex API Security Change)**
- Handle Plex removing shared user tokens from the API ([#123](https://github.com/StudioNirin/PlexCache-R/issues/123))
- Add `switchHomeUser()` fallback for home/managed users without tokens
- Add manual token paste for shared users in Settings → Users
- Add self-service auth link (`/auth/link`) — shared users sign in via Plex OAuth to link their token
- Add Plex database direct read fallback — queries the Plex SQLite DB for shared users with no token (TV next-episode resolution with season boundary handling, partially watched movies)
- New `core/plex_db.py` module with configurable database path in Settings → Users
- Note: Plex [announced](https://forums.plex.tv/t/information-related-to-security-vulnerabilities/935164/57) they are reverting the shared token removal, but plan to remove them again in the future

**Authentication**
- Add authentication system with Plex OAuth and sliding sessions
- Session persistence across container restarts
- Configurable session duration with remember-me support

**Extension-Free Caching**
- Cache all associated files (artwork, NFOs, subtitles, etc.) alongside video files
- Configurable associated file support with show-root file discovery
- Prevents array spinups from Plex reading metadata while video is cached

**Multi-Version (4K) Media Support**
- Cache multiple versions of the same media item (e.g., 1080p + 4K)
- Sibling file grouping for multi-version directories

**Dashboard & UI Improvements**
- Fix dashboard watchlist/OnDeck cached counts to show correct values
- Add cache limit marker and info line to dashboard cache drive card
- Improve maintenance UI with bulk actions and directory grouping
- Improve log viewer with clickable level filters, smart auto-scroll, and regex search
- Unify associated files display with reusable macros and viewport-aware popups
- Add connection-lost detection to operation banner
- UI refinements and sidebar version display

**Duplicate Detection**
- Add duplicate file detection with multi-version awareness
- Ignore/dismiss functionality for duplicate scan items
- Expanded detail rows with visual nesting

**Library Management**
- Improve library path mapping UX for multi-location libraries
- Consolidate library path edit into single form with bulk save
- Sync library path mappings with Plex on toggle

### v2.0 Updates (December 2025)

- **Fix double path conversion bug** ([#28](https://github.com/StudioNirin/PlexCache-D/issues/28)): Paths already in `real_source` format are no longer double-converted when `plex_source` is set to `/`. This was causing `FileNotFoundError` crashes for users with certain multi-path configurations.

- **Graceful error handling for missing files** ([#27](https://github.com/StudioNirin/PlexCache-D/issues/27)): Script now skips inaccessible files instead of crashing. Logs a warning and continues processing remaining files.

- **Self-healing exclude list cleanup**: At startup, stale entries (files that no longer exist on cache) are automatically removed from the exclude list. This prevents orphaned entries from accumulating.

- **Prevent multiple instances**: A lock file (`plexcache.lock`) ensures only one instance of PlexCache can run at a time. If another instance is already running, the script exits with an error. Lock is automatically released on exit or crash.

- **Improved cache limit logging**: Now shows both total cache drive usage and PlexCache-tracked size separately, making it easier to understand cache utilization.

- **Fix RSS author lookup**: Plex changed the author ID format in RSS feeds from numeric ID to UUID (extracted from user thumb URL). Updated to store and lookup both formats for proper username attribution in RSS warnings.

- **Store user UUID in settings**: User entries now include `uuid` (extracted from Plex thumb URL) for debugging RSS author lookup issues. Run `--refresh-users` to populate for existing setups.

- **Unraid Mover Exclude File Update**: This file now allows for the user to enter custom exclusions above a marker line, that won't be altered or removed by Plexcache.
